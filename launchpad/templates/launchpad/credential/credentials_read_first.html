Get started with Credentials

Proceed cautiously, if you dont know what you are doing, Stop!

While public key cryptography is a well documented area of computing, we assume that you know the basics and understand the risks associated with storing keys on a public facing server. If you dont, get a professional to review / understand the implications.

While Aerobridge stores the keys with simple security which should suffice for simpler non-production applications. For production use, we recommend using a system such as Hashicorp Vault to store / manage secrets, we can help you with the setup, see the contact form below:



  1. Create a public / private key for you any pilot / operator
  2. You can use a following commands to create a public / private key pair.

    ssh-keygen -t rsa -b 4096 -m PEM -f operator.key
    openssl rsa -in operator.key -pubout -outform PEM -out operator.key.pub

  3. The above code will generate a public key in PEM format and a private key operator.key. These are confidential files and can then be stored in Aerobridge via the Tokens Manager. Private keys should not be stored in the server.
