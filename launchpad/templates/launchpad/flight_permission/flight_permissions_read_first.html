Get started with the Aerobridge trusted flight system

Aerobridge gives you the ability to issue cryptographically signed tokens for authorized GCSs, to ensure that only authorized flights can be launched in the drone. You can think of this as OTPs. The following is the flow for getting / managing flight permission artefacts:



  1. Create a Flight Plan in the GCS and upload a co-ordinates file as KML in Aerobridge
  2. In Aerobridge, create a Flight Operation, assign a flight plan to a operation and a drone, you will have to input additional fields around the operation (or you can use the GCS).
  3. Flight permissions are generated automatically and you can see them Flight Operations menu.
  4. Once the flight permission isssued, you can see the permission artefact at in the Flight Permissions menu.
  5. After the operation is complete the signed flight logs from the drone can be uploaded to the server.
